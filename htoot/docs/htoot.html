 Description:  Hunchentoot is a web server written in Common Lisp and at
   the same time a toolkit for building dynamic websites. As a stand-alone
   web server, Hunchentoot is capable of HTTP/1.1 chunking (both directions),
   persistent connections (keep-alive), and SSL.
installed? : () -> Boolean 
 installed?() is true if the symbol :HUNCHENTOOT is in *features*.
 We then assume, that Hunchentoot is installed (optimistic :)
start : (NNI,STR,STR) -> SEX 
 a:=start(port,address,docroot) starts the webserver listening on
 port 'port' and address 'address'. Static files are served from
 docroot (a path). The return value 'a' is an instance off the acceptor
 class and should be stored in order to stop/restart the server.
stop : SEX -> SEX 
 stop(a) stops the webserver, where 'a' is the return value of the
 start function.
restart : SEX -> SEX 
 restart(a) starts the server with acceptor 'a' again after it has
 been stopped.
started? : SEX -> Boolean 
 started?(a) returns true if the server has beeen started.
getAddress : SEX -> SEX 
 getAddress(a) returns the ip address listening on.
getPort : SEX -> SEX 
 getPort(a) return the ip port listening on.
describe : SEX -> SEX 
 describe(a) returns all info about a (essentially the CL
 inspect/describe function)

 folderDispatch("/docs/","/home/kfp/devel/fricas-book/") creates and
 pushes a folder dispatcher and handler, so that for example:
 http://localhost:8080/docs/index.html displays the file index.html.
 Note: the end slashes in both parameters are required (acc. to docu)
