mkVECT : (NNI,NNI,NNI,LI,LNI,LLF,LLF) -> STR 
 mkVECT(np,nv,nc,nvp,ncp,v,c) creates a VECT OOGL format string, where
 np: number of polylines, nv: number of vertices, nc: number of colors
 nvp: list of numbers of vertices per polyline
 ncp: list of numbers of colors per polyline (0,1 or #vert)
 v : list of point coordinates, c: list of rgba values
 Ex: mkVECT(1,1,1,[1],[1],[[1.0,2.0,3.0]],[[0.0,0.0,1.0,1.0]])
 is a blue point (pixel).
mkMESH : (LLF,LLN,LDF -> LDF,B,B) -> STR 
 mkMESH(pts,idx,colfun,false,false) creates a MESH OOGL format string,
 where pts: list of list of three double float coordinates
 idx: the indices of the vertices enhanced with color information by
 applying the 'colfun' function to each point. The last two boolean's
 indicate whether the mesh is wrapped in u and/or v direction
 respectively. Note that the mesh is always colored, i.e. it is a
 CMESH, CuMESH, CvMESH or CuvMESH.
mkSKEL : (NNI,NNI,LLF,LLN,LLF) -> STR 
 mkSKEL(nv,np,pts,idx,colors) creates a SKEL OOGL format string,
 where nv: number of vertices, np: number of polylines, pts: list
 of list of three double floats (coordinates), idx: the indices of
 each polyline and colors: the RGBA color of each polyline.
mkQUAD : (LLF,LLN,LDF -> LDF) -> STR 
 mkQUAD(pts,idx,colfun) creates a QUAD OOGL format string, where
 pts: list of list of double floats and idx: a list of list of four
 indices into the points. The colors are generated by
 the function 'colfun' applied to each point.
mkCOFF : (NNI,NNI,LLF,LLN,LLF) -> STR 
 mkCOFF(nv,np,pts,idx,colors) creates a COFF OOGL format string,
 where nv: number of vertices, np: number of faces, pts: list
 of list of three double floats (coordinates), idx: the indices of
 each face and colors: the RGBA color of each face.
mkOBJF : (LLF,LLN) -> STR 
 mkOBJF(pts,idx) creates a wavefronobject file with vertices v and
 faces f.
mkOBJL : (LLF,LLN) -> STR 
 mkOBJL(pts,idx) creates a wavefronobject file with vertices v and
 polylines l.
mkFITZ : (STR,SDF,DF,SDF,DF,NNI,LLF) -> STR 
 mkFITZ(id,x0..x1,dx,y0..y1,dy,nc,pts) creates a GLE Fitz format.
 FITZ fits smooth curves based on a set of 3D data points.
 A file named .dat will be created from 'pts', then a begin
 end fitz block written that can be sent to GLE which in turn
 generates the .z file for further processing.
